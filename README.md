# 🎮 自作GAME STREAM DJ v1.5.8-beta
配信者向け高機能BGM管理ツール。YouTube音源をコピーするだけで曲を登録、自動でループ解析、OBS連動など自動曲を切り替え機能

## 🚀 起動方法

1. **`start.bat`** をダブルクリック
2. ブラウザで `http://127.0.0.1:8000` を開く

## v1.5.8-beta (Current)

### 機能改善・安定性向上
- **フェード不発バグの完全修正**: 
    - Restartモードで曲を最初から開始する際、動画再生状態が維持されたまま音が鳴り続けてしまうYouTube APIの仕様に対応し、切り替え前に明示的な停止処理を追加しました。
    - OBS等から一瞬で複数の切り替え信号が飛んだ際（多重切り替え）に、フェード処理が競合して暴走する問題を「フェードID」管理で回避。複雑な連動時でも確実にスムーズなフェードが行われます。
- **UIエンジン負荷の可変最適化**: シークバーやループ位置の監視を、これまで常に「1秒間に20回」行っていましたが、「普段は1秒間に4回に落とし、Deck Bのループ終点3.5秒前に迫った瞬間だけ本来の精密さ（1秒間に20回）にブーストする」可変エンジンペースを導入しました。これにより**ブラウザ（CPU）にかかる負荷を約5分の1に激減**させました。
- **OBSの自動バックグラウンド再接続**: アプリ起動後、OBS側が後から立ち上げられた場合や再起動された場合でも、**バックグラウンドで10秒ごとに静かな生存確認を送り、OBS復帰時に完全自動で（ログを汚さずに）再接続**するようになりました。

## v1.5.7-beta
- **空デッキ時の再生防止修正**: デッキを「No Track」状態にしているにも関わらず、デッキ切り替え（SWITCH）を行うと以前の曲が再生されてしまう不具合を修正しました。

## v1.5.6-beta

### 機能改善
- **デッキ切り替え再生タイミングの最適化**: 
    - 「Restart (0s)」または「Next (Nxt)」モードで曲を最初から開始する際、これまではボタンを押した瞬間に再生が始まっていましたが、**クロスフェード（フェードアウト/イン）が完了した後に再生が始まる**ように改善しました。
    - 前の曲が完全に消えてから新しい曲が立ち上がるため、より自然でプロフェッショナルな切り替えが可能になります。
    - ※ 「Resume (Res)」モードは、これまで通りボタンを押した瞬間に再生位置が同期されます。

## v1.5.5-beta

### 機能追加・UI改修
- **名前変更のキャンセル**: 曲名やグループ名をダブルクリックして編集している際、Escキーを押すことで編集を破棄して元の名前に戻せるようになりました。
- **ループ分析キャンセル**: 分析中（⏳ Analyzing...）の際に出る「✖」ボタンで、処理を中断して待機状態に戻せるようになりました。
- **クリップボード監視の「自動分析」統合**: 設定項目に「Deck B (L+Anlyz)」などが登場。コピーするだけで「デッキへのロード ＋ 分析」ワンステップで完了します。
- **クリップボード監視の個別ホットキー (F2 / F9)**: アプリが裏に隠れていても、F2で監視開始、F9で停止が瞬時に行えます。ブラウザの操作を邪魔しない独立したキー設定です。
- **バックグラウンド解析の強化**: ウィンドウが非アクティブな状態でも、URLをコピーした瞬間に解析を開始するようになりました（直接デッキにロードする場合）。作業を中断せずに次の曲の準備が整います。
- **重複管理の階層限定化**: 重複警告を「追加先の階層」のみに限定しました。これにより、別のグループに同じ曲が存在していても、現在の場所になければスムーズに追加できます。
- **内外部クリップボードの防護**: アプリ内でCtrl+CをしてOS側にURLをコピーした際、自身の監視機能が反応して同じ曲を重複登録してしまうのを自動で無視するようにしました。

### 仕様変更
- **Deck Load時のライブラリ追加の廃止**: クリップボード監視のターゲットを「Deck A/B (Load)」系にした際、以前はバックアップとして一旦ライブラリにも追加されていましたが、これを廃止し「デッキにのみロード」するよう変更しました。ライブラリを汚さずにさっと流すことができます。

## 🎥 OBS Sync (自動シーン切り替え)

デッキの切り替えに連動して、OBS のシーンを自動で変更できます。

### セットアップ手順
1.  **OBS側の設定**:
    - [ツール] -> [WebSocket サーバー設定] を開く。
    - [WebSocket サーバーを有効にする] をチェック。
    - **サーバーポート** (`4455`) と **サーバーパスワード** を確認・メモします。
2.  **本ツール側の設定**:
    - ブラウザで **Switch Triggers (⚙️)** を開く。
    - **OBS接続設定** セクションに、Host / Port / Password を入力。
    - **「接続テスト」** ボタンを押して接続確認。ステータスが `Connected` になればOK。
    - **Scene A / Scene B** に、OBSで作成したシーン名を正確に入力（例: `BGM_A`, `BGM_B`）。
3.  **A→B / B→A 切り替え設定**:
    - 各方向のセクションで **「OBS連動」** をチェックします。
4.  **動作**:
    - **アプリ → OBS**: `SWITCH DECK` を行うと、チェックが入っている方向の切り替え時に指定シーンへ瞬時に切り替わります。
    - **OBS → アプリ**: OBS側で指定シーン（Scene A / B）に切り替えると、自動的に対応するデッキへ切り替わります（双方向連動）。

> **Note**: `.env` ファイルは不要です。全てブラウザのSettings画面から設定・保存できます。

## ⌨️ キーボードショートカット

| キー | 動作 |
|------|------|
| **Space** | アクティブなデッキの再生/一時停止 |
| **Switch Key** | デッキ切り替え（Switch Triggersで任意のキーを設定可能。グローバル対応）。`F8`がデフォルト。 |
| **Ctrl + Z** | 直前のライブラリ操作などの取り消し (Undo) |

> **重要**: ブラウザの制限により、**このアプリのタブがアクティブでない（別タブを見ている）場合、キーを押しても反応が遅延します**。
> タブに戻った瞬間に切り替えが実行されます。
> 別ウィンドウとして切り離して表示している場合や、アプリのウィンドウがアクティブな場合は、他のアプリ操作中でも即座に反応します。

## ⚠️ 注意事項・ヒント

- **自動ループ解析 (Analyze)**: YouTubeの仕様変更により失敗する場合は、`pip install -U yt-dlp` で最新版に更新してください。
- **データ保存**: 設定やライブラリは `backend/dj_state.json` に自動保存されます。バックアップはこのファイルをコピーしてください。
- **Node.js**: PCに Node.js がインストールされていると、解析の精度と速度が向上します。
- **F8ホットキー**: ブラウザがバックグラウンド（非表示タブ）の場合、ブラウザの仕様によりタブを表示した瞬間に切り替えが実行されます。
